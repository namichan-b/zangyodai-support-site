# CSS配置ガイド - 残業代サポートサイト

## 🎯 現状の問題
- 追加CSSが4777行と巨大すぎる
- すべてカスタマイザーに入れると管理が大変
- ページの読み込みが遅くなる可能性

## ✅ 推奨する配置方法

### 1. 各ページのCSS欄に配置（最優先）

#### メリット
- ページごとに管理できる
- 不要なCSSを読み込まない
- メンテナンスが楽

#### 設定方法
1. 固定ページ編集画面を開く
2. 右サイドバーの「カスタムCSS & JS」
3. 「CSS用コード」欄にペースト
4. 更新

### 2. 共通部分のみカスタマイザーに配置

#### 対象となるCSS（約100行程度に削減）
```css
/* ============================================
   全ページ共通の基本設定
   ============================================ */

/* カラー設定 */
:root {
    --main-color: #2874a6;
    --accent-color: #c9a961;
    --text-color: #333333;
}

/* フォント設定 */
body {
    font-family: "Yu Gothic", "游ゴシック", YuGothic, "游ゴシック体", sans-serif;
    color: var(--text-color);
}

/* ボタン共通スタイル */
.cta-button {
    background: linear-gradient(135deg, #c9a961 0%, #d4af37 100%);
    color: white;
    padding: 15px 30px;
    border-radius: 25px;
    font-weight: bold;
    transition: all 0.3s;
}

/* レスポンシブ基本設定 */
@media (max-width: 768px) {
    .pc-only { display: none; }
}
@media (min-width: 769px) {
    .sp-only { display: none; }
}
```

## 🚀 移行手順

### STEP 1: CSSを分割
1. 追加cssまとめ.cssを開く
2. ページごとにCSSを分割
3. 各ページ用のCSSファイルを作成

### STEP 2: 各ページに配置
1. WordPress管理画面にログイン
2. 固定ページを編集
3. カスタムCSS & JS欄にペースト
4. 更新

### STEP 3: カスタマイザーのCSSを最小化
1. 外観 → カスタマイズ → 追加CSS
2. 共通部分のみ残す（100行程度）
3. 公開

---

## ⚠️ 注意事項

### やってはいけないこと
- すべてのCSSをカスタマイザーに入れる
- 使わないCSSを残す
- !importantの乱用

### パフォーマンス改善のコツ
- 不要なCSSは削除
- 似たスタイルは統合
- メディアクエリはまとめる
