## ステップ1: CSSの追加（全ページで1回のみ）

まず、サイト全体に適用されるCSSを追加します。

### 手順：

1. WordPress管理画面にログイン
2. 左メニューから **「外観」→「カスタマイズ」** をクリック
3. カスタマイザー画面の左メニューから **「追加CSS」** をクリック
4. 既存のCSS（他のページ用）の **一番下** までスクロール
5. 最後に、以下のコメントを追加：

```css
/* ============================================
   解決事例ページ用CSS
   ============================================ */
```

6. `10_解決事例_CSS.css` ファイルを開く
7. **CSS全体をコピー**（Ctrl+A → Ctrl+C または Cmd+A → Cmd+C）
8. WordPressの「追加CSS」エリアに **ペースト**（Ctrl+V または Cmd+V）
9. 右上の **「公開」** ボタンをクリック

これでCSSの設定は完了です。

## ステップ2: 新規ページの作成

次に、「解決事例」ページを作成します。

### 手順：

1. WordPress管理画面で、左メニューから **「固定ページ」→「新規追加」** をクリック
2. ページタイトル欄に以下を入力：
   ```
   解決事例
   ```

## ステップ3: HTMLコードの貼り付け

カスタムHTMLブロックを使って、作成済みのHTMLコードを貼り付けます。

### 手順：

1. ブロックエディタ画面で、左上の **「＋」ボタン** をクリック
2. 検索ボックスに **「カスタムHTML」** と入力して検索
3. **「カスタムHTML」ブロック** を選択（黒い背景のアイコン）
4. `10_解決事例.txt` ファイルを開く
5. **HTML全体をコピー**（Ctrl+A → Ctrl+C または Cmd+A → Cmd+C）
6. WordPress編集画面の **カスタムHTMLブロック** 内に **ペースト**（Ctrl+V または Cmd+V）

## ステップ4: プレビューで確認

HTMLを貼り付けたら、見た目を確認します。

### 手順：

1. 右上の **「プレビュー」** ボタンをクリック
2. **「新しいタブでプレビュー」** を選択
3. デザインが正しく表示されているか確認

### 確認ポイント：

✓ ページヒーローがグラデーションブルーで表示されている
✓ フィルターボタンが横並びで表示されている
✓ 事例カードが2つ表示されている
✓ 各事例の回収金額が大きく表示されている
✓ ケースの概要、解決のポイントが正しく表示されている
✓ CTAボタン（お問い合わせフォーム）が正しくリンクされている
✓ 最後のCTAセクションがオレンジのグラデーションで表示されている
✓ スマホサイズでも正しく表示される

## ステップ5: ページ設定

公開前に、ページの基本設定を行います。

### 手順：

1. 右側のサイドバーで **「ページ」タブ** を選択
2. **「URL」** を設定（例：`cases` または `kaiketsujire`）
   - わかりやすく、SEOに配慮したURLにします
3. **「親ページ」** を設定（必要に応じて）
4. **「抜粋」** を入力（検索結果やリンクに表示される概要文）：
   ```
   残業代サポートが解決した実際の事例をご紹介。ルート営業640万円回収、運送業ドライバー300万円回収など、あなたと似た状況の事例があれば参考にしてください。
   ```

## 今後の事例追加方法

新しい解決事例を追加する場合は、以下の手順で行います。

### 手順：

1. **カスタムHTMLブロック** を開く
2. 既存の事例（`<article class="case-card-detailed">`）をコピー
3. 適切な位置にペースト
4. 以下の内容を変更：
   - バッジの文言
   - 事例のタイトル
   - メタ情報（年齢、業種、解決期間）
   - 回収金額
   - 詳細グリッドの内容
   - ケースの概要
   - 解決のポイント

### 注意事項：

⚠️ **新しい事例を追加する際は、必ず以下を確認してください：**

1. **実際のケースに基づいている**（架空の事例は掲載禁止）
2. **依頼者が特定されない**（年齢、地域、業種を曖昧にする）
3. **結果を保証しない**（「必ず」「確実に」等の表現は使わない）
4. **誇大表現を避ける**（「圧倒的」「最高」等は使わない）

### 事例のハイライト表示：

特に注目させたい事例には、ヘッダーに `highlight` クラスを追加します：

```html
<!-- ハイライト表示する場合 -->
<div class="case-card-header highlight">

<!-- 通常表示の場合 -->
<div class="case-card-header">
```

## トラブルシューティング

### デザインが反映されない場合

1. **CSSが正しく追加されているか確認**
   - 「外観」→「カスタマイズ」→「追加CSS」
   - 解決事例ページ用のCSSコードが貼り付けられているか確認
   - 「公開」ボタンを押したか確認

2. **キャッシュをクリア**
   - ブラウザのキャッシュをクリア（Ctrl+Shift+Del または Cmd+Shift+Del）
   - WordPressのキャッシュプラグインを使っている場合は、そちらもクリア

3. **HTMLブロックを確認**
   - カスタムHTMLブロックが正しく選択されているか
   - HTMLコードが全てコピーされているか

### レイアウトが崩れる場合

1. **SWELLテーマの設定を確認**
   - 「外観」→「カスタマイズ」→「コンテンツの設定」
   - 「記事・固定ページ」の設定で、不要な装飾がオフになっているか確認

2. **ブロックの順序を確認**
   - カスタムHTMLブロックが1つだけで、他のブロックと混在していないか確認

### フィルターボタンが動作しない場合

現在のバージョンでは、フィルターボタンは**見た目のみ**で、実際の絞り込み機能はありません。

今後、事例が増えてきたら、JavaScriptを追加してフィルター機能を実装できます。

### 回収金額の位置がずれる場合（PC表示）

CSSの `.case-amount-large` クラスが正しく適用されているか確認してください。

スマホでは自動的に下部に移動するようになっています。

## 次のステップ

ページが公開できたら、以下を実施しましょう：

### 1. **メニューに追加**
- 「外観」→「メニュー」
- 「解決事例」ページをメインメニューに追加
- 適切な順序に並び替え（推奨：トップページの次）

### 2. **トップページからリンク**
- トップページに「解決事例」へのリンクを追加
- CTAボタンや案内文から誘導

### 3. **内部リンクの設置**
- 他のページ（相談の流れ、費用・料金など）から「解決事例」へリンク
- 特に「あなたと似た状況の事例はこちら」等の文言で誘導

### 4. **スマホでの表示確認**
- 実際のスマートフォンで表示を確認
- ボタンのタップが正しく動作するか確認
- 回収金額が下部に移動しているか確認

### 5. **アクセス解析の設定**
- Google Analyticsで「解決事例」ページの閲覧を追跡
- CTAボタンのクリック率を計測（イベントトラッキング）

## よくある質問

### Q. CSSは毎回追加する必要がありますか？

A. いいえ、CSSは最初の1回だけ追加すればOKです。次回以降、事例を追加する場合も、HTMLを修正するだけで済みます。

### Q. 事例は何件まで掲載できますか？

A. 技術的には無制限ですが、**多すぎると読者が迷います**。5〜10件程度が適切です。それ以上増える場合は、カテゴリ分けやフィルター機能の実装を検討してください。

### Q. 架空の事例を掲載したい場合は？

A. **架空の事例は原則として掲載できません**。どうしても掲載する場合は、「※これは一般的なケースを想定した架空の事例です」と明記する必要があります。ただし、実際の事例のみを掲載することを強く推奨します。

### Q. 回収金額を大きく見せたい場合は？

A. **誇大広告は絶対に避けてください**。実際の回収金額をそのまま記載してください。また、「満額回収」等の表現は、過度な期待を抱かせる可能性があるため、慎重に使用してください。

### Q. フィルター機能を実装したい場合は？

A. 現在のバージョンでは見た目のみです。JavaScriptを追加すれば実装できますが、事例が少ないうちは不要です。事例が10件を超えたら、Claude Codeに相談してフィルター機能を実装しましょう。

### Q. 事例の順序を変更したい場合は？

A. HTMLコード内の`<article class="case-card-detailed">`ブロックの順序を入れ替えることで変更できます。一般的には、**回収金額が大きい事例**や**最近の事例**を上位に配置することが効果的です。
