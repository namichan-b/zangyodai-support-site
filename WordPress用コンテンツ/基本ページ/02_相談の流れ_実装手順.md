# 「相談の流れ」ページ - SWELL実装手順書

## 📋 概要

このドキュメントは、モックアップデザインを WordPress の SWELL テーマで実装するための手順書です。

**作成するページ:** 相談の流れ
**ページタイプ:** 固定ページ
**使用ファイル:**
- `02_相談の流れ_SWELL実装版.html` - ページ本文（カスタムHTMLブロック用）
- `02_相談の流れ_カスタムCSS.css` - スタイルシート（追加CSS用）

---

## 🎯 実装の3ステップ

### ステップ1: カスタムCSSを追加

1. WordPress管理画面にログイン
2. **外観 → カスタマイズ** をクリック
3. **追加CSS** をクリック
4. `02_相談の流れ_カスタムCSS.css` の内容をコピー&ペースト
5. **公開** ボタンをクリックして保存

**または、SWELL設定から追加する場合：**
1. WordPress管理画面にログイン
2. **SWELL設定 → エディター設定** をクリック
3. **カスタム書式** タブを選択
4. `02_相談の流れ_カスタムCSS.css` の内容をコピー&ペースト
5. **変更を保存** をクリック

---

### ステップ2: 固定ページを作成

1. WordPress管理画面にログイン
2. **固定ページ → 新規追加** をクリック
3. **ページタイトル** に「相談の流れ」と入力

---

### ステップ3: コンテンツを追加

#### 3-1. カスタムHTMLブロックを追加

1. ブロックエディター画面で **+** ボタンをクリック
2. 検索ボックスに「カスタムHTML」と入力
3. **カスタムHTML** ブロックを選択
4. `02_相談の流れ_SWELL実装版.html` の内容を**全てコピー**
5. カスタムHTMLブロックに貼り付け

#### 3-2. ページ設定

**右サイドバーの設定:**

1. **パーマリンク（URL）を設定**
   - URLスラッグ: `soudan-flow` または `consultation-flow`

2. **アイキャッチ画像を設定（任意）**
   - 相談の流れのイメージ画像を設定

3. **SWELL設定（固定ページごとの設定）**
   - サイドバーの表示: **表示しない**
   - パンくずリスト: **表示する**
   - 記事下エリア: お好みで設定

4. **SEO設定（Yoast SEO または Rank Math 使用時）**
   - メタディスクリプション: 「残業代請求のご相談から解決までの流れをご説明します。初回相談無料、着手金0円。福岡発・全国対応の弁護士による残業代請求サポート。」
   - フォーカスキーワード: 「相談の流れ 残業代」

---

### ステップ4: プレビュー確認と公開

1. **プレビュー** ボタンをクリックして表示を確認
2. 以下の点をチェック：
   - ✅ ステップフローが正しく表示されているか
   - ✅ ステップ番号の色（1番は緑、2-6番はオレンジ）
   - ✅ 必要書類セクションのカードが4列表示されているか
   - ✅ FAQのアコーディオンが動作するか
   - ✅ CTAボタンのリンクが正しいか
   - ✅ スマホ表示も確認（レスポンシブ対応）

3. 問題がなければ **公開** をクリック

## 🎨 デザインの特徴

### 1. ページヒーロー
- **グラデーション背景**: ダークブルー（#04384c → #0a5a7a）
- **グリッドパターン**: 背景に透過グリッド
- **白文字**: 視認性の高い大きな見出し

### 2. ステップフロー
- **番号付き円**: ステップ1は緑、ステップ2-6はオレンジ
- **縦線**: ステップ間を繋ぐグラデーション線
- **詳細ボックス**: グレー背景に左側ブルーのアクセント
- **チェックマーク**: 緑色の✓でリスト項目を強調

### 3. 必要書類セクション
- **4列グリッド**: PC表示で4つのカードを横並び
- **アイコン**: 絵文字を使用（📄⏰🚚🏢）
- **ホバー効果**: カードが浮き上がるアニメーション
- **特別なケース**: オレンジ枠の注意喚起ボックス

### 4. FAQセクション
- **アコーディオン**: クリックで開閉
- **+アイコン**: クリック時に45度回転してXになる
- **ホバー効果**: カードにシャドウが追加

### 5. CTAセクション
- **オレンジグラデーション**: 目立つ背景色
- **白いボタン**: コントラストの高いCTA
- **電話番号**: 大きく太字で表示

## 🔧 カスタマイズ方法

### 色を変更したい場合

**メインカラー（ダークブルー）の変更:**
```css
/* 変更前 */
background: linear-gradient(135deg, #04384c 0%, #0a5a7a 100%);

/* 変更例: 紺色に変更 */
background: linear-gradient(135deg, #1e3a8a 0%, #3b82f6 100%);
```

**ステップ番号の色変更:**
```css
/* ステップ1（緑色） */
.step-number-1 {
    background: linear-gradient(135deg, #28a745 0%, #20c997 100%) !important;
}

/* ステップ2-6（オレンジ色） */
.step-number-2,
.step-number-3 {
    background: linear-gradient(135deg, #ff6b35 0%, #ff8964 100%);
}
```

### ステップ番号のサイズ変更

```css
.step-number {
    width: 80px;   /* 変更: 例えば100pxに */
    height: 80px;  /* 変更: 例えば100pxに */
    font-size: 32px; /* 変更: 例えば40pxに */
}
```

### グリッドの列数を変更

```css
.documents-grid {
    /* 4列から3列に変更 */
    grid-template-columns: repeat(3, 1fr);
}
```

## 🆘 トラブルシューティング

### Q1: CSSが反映されない

**原因:**
- ブラウザのキャッシュが残っている
- CSS の記述ミス

**解決方法:**
1. ブラウザのキャッシュをクリア（Ctrl+Shift+R または Cmd+Shift+R）
2. WordPress のキャッシュをクリア（SWELL設定 → キャッシュクリア）
3. CSS に構文エラーがないか確認

### Q2: FAQのアコーディオンが動作しない

**原因:**
- JavaScript が読み込まれていない
- onclick属性が削除されている

**解決方法:**
1. HTMLコードを確認し、`onclick="this.parentElement.classList.toggle('open')"` が含まれているか確認
2. ブラウザのコンソールでエラーを確認

### Q3: スマホで横スクロールが発生する

**原因:**
- 要素の幅が画面からはみ出ている

**解決方法:**
1. レスポンシブCSSが正しく適用されているか確認
2. 以下のCSSを追加:
```css
body {
    overflow-x: hidden;
}
```

### Q4: ステップ番号の色が全て同じになる

**原因:**
- CSSの詳細度の問題
- クラス名のミス

**解決方法:**
1. HTML のクラス名を確認: `step-number-1`, `step-number-2`, etc.
2. CSS に `!important` を追加:
```css
.step-number-1 {
    background: linear-gradient(135deg, #28a745 0%, #20c997 100%) !important;
}
```
